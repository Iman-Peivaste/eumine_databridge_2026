🎓 KEY CONCEPTS (Tutor Edition)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Why conda + pinned versions?
  → conda manages PyTorch CUDA dependencies automatically
  → Pinned versions (torch==2.4.0, not torch>=2.0) prevent surprises
  → Your model was trained with torch 2.4.0 — must use same version

Why verify.py?
  → Catches 90% of "it doesn't work on this machine" errors BEFORE sprint
  → Tests import, load, predict, federation — full pipeline
  → If it passes, you're good. If it fails, you have time to fix.

Why Colab option?
  → Test without local CUDA setup (10 min vs 15 min)
  → Free GPU (T4) is fast enough for inference
  → Share link with team for testing
  → Practice before Cluj

Why federation interface?
  → Your model is just one piece. Partners have their own.
  → FederatedEnsemble learns how to weight each model
  → Runs on calibration data → predicts on test set
  → Winner: who makes the best ensemble 🏆

Issue: "conda: command not found"
  → Install miniconda: https://docs.conda.io/en/latest/miniconda.html
  → Then: bash deploy/install.sh

Issue: "No module named 'alignn'"
  → conda activate catallist_stage2
  → pip install alignn==2026.4.2

Issue: CUDA version mismatch
  → Use CPU installer: bash deploy/install_cpu.sh
  → (Or use Colab GPU)

Issue: "matfed_api not found"
  → install.sh tries to add it automatically
  → If it fails: cd ../hackathon_ref/matfed-api-template && pip install -e .

Issue: Disk space full (14.3 GB models)
  → Free up space or test on Colab instead

Issue: Model won't load
  → Run: python deploy/verify.py
  → It will tell you exactly what's missing

Issue: Forgot to test before Cluj
  → Open Colab notebook in 30 seconds
  → Test there
  → You're still fine 😅
